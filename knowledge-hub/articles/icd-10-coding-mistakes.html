Top 10 ICD-10 Coding Mistakes

These are the most common coding errors found in medical billing audits. Each has specific consequences and prevention strategies.

1. Using Unspecified Codes

Using unspecified or "NOS" (not otherwise specified) codes when more specific codes are available and documented.

Example: Coding "Essential hypertension, unspecified" (I10) when the documentation shows "stage 2 hypertension with chronic kidney disease" (I12.x)

Impact: Lower reimbursement, potential claim denials, compliance issues

2. Missing Laterality Indicators

Failing to include laterality (right, left, bilateral) when required by the code structure.

Example: Coding knee pain as M25.5 without specifying right or left knee (should be M25.51 or M25.52)

Impact: Immediate claim rejection, requires resubmission

3. Incorrect Principal Diagnosis Sequencing

Listing diagnosis codes in the wrong order, placing secondary conditions as primary.

Example: For an inpatient admission, listing Type 2 diabetes as primary instead of the acute condition that prompted admission (pneumonia)

Impact: Reduced DRG payment, audit red flags

4. Omitting Required Fifth Characters

Using incomplete codes that require seventh characters for specificity.

Example: Coding a fracture as S72.001 when documentation indicates it's a subsequent encounter (should be S72.001D)

Impact: Invalid codes, automatic claim rejection

5. Unbundling Services

Separating component codes that should be billed together as a bundled code.

Example: Billing office visit, exam, and assessment separately when a single established patient office visit code is appropriate

Impact: Claim denials, fraud accusations, forced refunds

6. Missing Etiology-Manifestation Relationships

Failing to code both the underlying cause and the secondary effect when required.

Example: Coding diabetic nephropathy without also coding the diabetes diagnosis

Impact: Incomplete clinical picture, denial of secondary condition reimbursement

7. Combining Codes That Should Be Separate

Using combination codes when documentation doesn't support all components of the combination.

Example: Using a hypertension and chronic kidney disease combination code when only hypertension is documented

Impact: Overstating severity, potential fraud allegations

8. Excluding Excludes1 Codes

Assigning two codes that have an "Excludes1" relationship in the ICD-10 guidelines.

Example: Coding both traumatic injury and sequela codes for the same condition in the same encounter

Impact: Technical denials, perceived coding fraud

9. Using Outdated or Deleted Codes

Submitting claims with ICD-10 codes that have been deleted or revised.

Example: Using 2023 codes in 2025 when codes have been updated with new specifications

Impact: Automatic claim rejection, processing delays

10. Coding Without Clinical Documentation

Assigning diagnosis codes that aren't supported by the clinical documentation.

Example: Coding diabetes when only being "ruled out" (R73.9 - abnormal glucose, not confirmed diabetes)

Impact: Compliance violations, audit findings, compliance penalties

Correct Code Sequencing

The order in which you list diagnosis codes matters. ICD-10 sequencing rules directly impact reimbursement levels and can trigger compliance audits.

Outpatient Visits (Office-Based)

List the primary reason for the visit first, followed by comorbidities and chronic conditions that affect treatment.

Correct: Acute bronchitis (J20), Asthma, Smoking history

Incorrect: Smoking history, Asthma, Acute bronchitis

Inpatient Hospital (DRG-Based)

List the condition that resulted in admission as the principal diagnosis first. This determines your DRG assignment.

Correct: Community-acquired pneumonia (J15), Type 2 diabetes, Hypertension

Incorrect: Type 2 diabetes, Hypertension, Community-acquired pneumonia
